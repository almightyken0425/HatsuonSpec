# 進度追蹤模組

## 職責

- 以題組為單位，記錄使用者在各題組內的累積答題歷史
- 供題組列表畫面計算並顯示練習進度
- 追蹤基準與刷題引擎的自適應權重系統完全獨立

---

## 進度計算公式

`history` 陣列最多 50 筆，尾端為最新答題，對應位置 50，往前遞減至位置 1。尚未填滿的槽位為 null，分子不計入。

- **分子:** Σ 位置編號 × 答題結果，僅計已答槽位
- **分母:** 1275，固定值，不隨 history 長度變化

---

## 資料結構

題組進度記錄與刷題引擎的字元紀錄使用相同的本地儲存層，以 `unit_id` 區分：

| 欄位 | 說明 |
|------|------|
| `unit_id` | 題組識別碼，格式與題組編號一致，例如 1-1、1-2 |
| `history` | 最近 50 筆答題結果，布林值陣列，`true` 代表答對，`false` 代表答錯 |
| `last_updated` | 最後更新時間戳 |

**`history` 滑動視窗設計：**

- 上限固定為 50 筆
- 新結果加入陣列尾端
- 超出上限時自動捨棄頭部最舊一筆

---

## 寫入時機

- **觸發:** 每次答題後，將答題結果寫入對應題組的 `history` 陣列
- **題組識別碼來源:** 當前 session 的來源
    - **IF 標準模式:** 識別碼為使用者選擇的題組編號
    - **IF 自由模式:** 識別碼為使用者選擇的題組編號
- **不寫入本模組的情境:**
    - 全字符混合
    - 聽選模式
    - 四選一

---

## 標準模式與自由模式的差異

- **標準模式:** 答題結果於 session 結束後寫入本地儲存，跨 session 累積
- **自由模式:** 答題結果於 session 結束後捨棄，不寫入本地儲存，不影響題組進度顯示

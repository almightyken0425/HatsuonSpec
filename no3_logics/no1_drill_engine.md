# 刷題引擎

## Session 流程

### 開始

- **觸發:** 使用者從題組選單進入刷題
- **載入:** 所選題組的字元清單
- **初始權重:** 從本地讀取過去答題紀錄算出

### 進行中

- 無限循環出題，答完一題馬上出下一題，直到使用者離開

### 結束

- **觸發:** 使用者按返回或離開畫面
- **執行:** 把這次 session 的答題結果存進本地儲存

---

## 出題邏輯

### 通用規則

- 選題從題組字元池按照各字的權重做加權隨機抽取
- 上一題出過的字這題不會再出
- 選項順序每題隨機排列，正確答案位置不固定
- **IF 題組字元不足：** 從整個主線字元池補齊選項

---

### 出題策略

#### 清音、濁音、半濁音

- **選題：** 從題組字元池加權隨機抽取
- **字卡顯示：** 單一假名字元
- **選項：** 各字的 romaji

#### 拗音

適用於清音拗音、濁音拗音、半濁音拗音。

- **選題：** 從題組字元池加權隨機抽取
- **字卡顯示：** 拗音假名字串，例如 きゃ、ぎゃ、ぴゃ
- **選項：** 各字的 romaji

#### 複合字元

適用於促音與長音符。字卡顯示複合字串，特殊字元以強調色標示；選項為動態計算的 romaji 字串。

**促音**

- **`kana1`：** 從同假名類型的清音池加權隨機抽取任意清音
- **`kana2`：** 從同假名類型的清音池加權隨機抽取首子音為子音的清音，排除あいうえお行與ん
- **字卡顯示：** `kana1` + っ/ッ + `kana2`，例如 きった
- **正確答案：** `kana1` romaji 接上 `kana2` 首子音重複再接 `kana2` romaji，例如 kitta

**長音符**

- **`kata1`：** 從片假名清音池加權隨機抽取任意清音
- **`kata2`：** 從片假名清音池加權隨機抽取任意清音，排除 `kata1`
- **字卡顯示：** `kata1` + ー + `kata2`，例如 コーヒ
- **正確答案：** `kata1` romaji 接上 `kata1` 尾音重複再接 `kata2` romaji，例如 koohi

---

### 干擾策略

一題決定一種策略，預設各半機率，可調整為僅段或僅行。該題所有干擾項均使用相同策略，加權隨機抽取，不重複且不得等於正確答案。

- **段策略：** 從末尾母音相同的字裡選
- **行策略：** 從首子音相同的字裡選
- **IF 候選不足：** 從整個字元池補齊

#### 清音、濁音、半濁音

- **段策略候選：** 末尾母音相同的字，a / i / u / e / o 各段
- **行策略候選：** 首子音相同的字，同行

#### 拗音

- **段策略候選：** 末尾音相同的拗音，ya / yu / yo 各段，候選數量充足
- **行策略候選：** 首子音相同的拗音，例如 きゃ 的首子音為 k，同行為 きゃ / きゅ / きょ 共三字；4選1 時候選上限為兩個，由整個字元池補齊

#### 複合字元

複合字元的干擾策略為選擇一個變數字元，一題決定一個變數，所有干擾項使用相同變數。

**促音**

- **換 `kana1` 策略：** 以 `kana1` 末尾母音為段元素，換 `kana1`，`kana2` 不動，例如 itta、ritta、mitta
- **換 `kana2` 策略：** 以 `kana2` 首子音為行元素，換 `kana2`，`kana1` 不動，例如 kitte、kitto
- 兩種策略各有 50% 被選到的機率

**長音符**

- **換 `kata1` 策略：** 以 `kata1` 末尾母音為段元素，換 `kata1`，`kata2` 不動
- **換 `kata2` 策略：** 以 `kata2` 首子音為行元素，換 `kata2`，`kata1` 不動
- 兩種策略各有 50% 被選到的機率

## 自適應權重

### 追蹤單位

- 追蹤基準為字元本身加上其所有構成元素
- 每次答題後，字元及所有構成元素的紀錄同步更新
- ぎゃ 的追蹤單位為 ぎゃ 整體、基底 き、dakuten、small_ya，以此類推
- 構成元素識別碼的完整規則詳見 component 規格

### 計算方式

- **`history`:** 每個追蹤單位最近 50 筆答題結果，視為固定 50 個位置的滑動窗口，最新的一筆對應位置 50，往前遞減至位置 1
- **已答槽:** 答對貢獻位置編號，答錯貢獻 0
- **空槽:** 每格貢獻位置編號乘以 0.5
- **分母:** 固定為 1275，即 1 加到 50 的總和
- **單位權重:** 1 減去加權正確率
- **字元最終出題權重:** 取所有構成單位中最高的單位權重

### 最低保護

- 任何單位的權重下限為 0.1

### Session 內即時更新

- **觸發:** 每次答題後
- **執行:** 立即重算所有相關字元的權重
- **計算:** session 內的臨時紀錄與歷史紀錄合併

---

## 答題紀錄格式

| 欄位 | 說明 |
|------|------|
| `unit_id` | 這個追蹤單位的 ID |
| `history` | 最近 50 筆答題，true 為答對、false 為答錯 |
| `last_updated` | 最後更新時間 |

- `history` 為滑動窗口，新結果加到尾端，超過 50 筆就把最舊的丟掉
